# 結晶対称性パズル 仕様書 v3

## アプリの目的

結晶点群の対称性を直感的に学ぶための学習ツール。
与えられた結晶点群から「商群（組成因子）」を一つずつ選んで取り除くことで、
群がどんな対称操作から構成されているかをジョルダン・ヘルダーの定理的に理解する。

正解・不正解の判定はなく、自由に試行錯誤できる学習ツールとして動作する。

---

## 数学的な仕様

### 基本的な考え方

- 対象：32種類の結晶点群（まず代表的なものから実装）
- ユーザーが選ぶのは「商群 G/N」（例：Cs, C3, C2 など）
- プログラム側が自動で「正規部分群 N」を特定し、構造を N に単純化する
- 商群は既約（それ以上分解できない）なもののみ提示する
  - 具体的には位数が素数の巡回群（Cp）または最小の非可換群のみ
- 分解が終わると恒等元Eだけの群になる
- 途中で選んだ商群の列が「組成因子」となる

### C3v の例

```
C3v
├── ユーザーが Cs（鏡映）を選ぶ
├── プログラムが正規部分群 C3 を自動特定
├── アニメーション：原子が鏡映面に対して折り返す動きをしてから、鏡映面が消える
├── 構造が C3v → C3 に変化
│
C3
├── ユーザーが C3（3回回転）を選ぶ
├── プログラムが正規部分群 E を自動特定
├── アニメーション：原子が120度回転する動きをしてから、回転軸が消える
├── 構造が C3 → E に変化
│
E（終了）
└── 組成因子：Cs, C3 が判明 → C3v は鏡映と3回回転から構成される
```

### 実装する点群（初期セット）

- C1, Ci, Cs（位数1〜2）
- C2, C2v, C2h（位数2〜4）
- C3, C3v, C3i（位数3〜6）
- C4, C4v, C4h, S4（位数4〜8）
- D2, D2h, D3, D3h, D3d（位数4〜12）
- T, Td, Oh（高対称群）

## 各エリアの詳細仕様

### 3D ビューア（左側メイン）

**表示スタイル：ワイヤーモデル**
- 原子：色付きの球体（元素ごとに色を変える）
- 結合：細い線
- 背景：ダークグレーまたは黒（結晶が見やすいように）

**対称要素の表示：ホバー・選択時のみ強調表示**
- 通常時：対称要素は表示しない
- ユーザーが商群リストのボタンにホバーしたとき：
  - 該当する対称要素を強調表示
  - 鏡映面：青い半透明の板
  - 回転軸：オレンジの矢印付きの輝く線
- ホバーが外れると対称要素は非表示に戻る

**操作**
- マウスドラッグ：3D回転
- スクロール：ズームイン・アウト
- キーボード操作でも全て完結できるようにする（左クリック依存にしない）

### アニメーション仕様（重要）

商群を適用したときのアニメーションは以下の順序で再生する：

**ステップ1：対称要素の強調（0.3秒）**
- 取り除く対称要素（鏡映面・回転軸など）が光って強調される

**ステップ2：対称操作の実行（0.6秒）**
- 対称操作を実際に実行するように原子が動く
- 鏡映（Cs）：原子群が鏡映面に対して折り返す動き
- n回回転（Cn）：原子群が360/n度回転する動き
- 反転（Ci）：原子群が反転中心を通って反転する動き
- 回反（Sn）：回転＋反転の複合動作

**ステップ3：対称要素の消去（0.3秒）**
- 動き終わった後、対称要素がフェードアウトして消える
- 構造が新しい（単純化された）群の形に落ち着く

アニメーションの速度は設定で変更可能（速い・普通・遅い）。

### 結晶構造データ：2モード

スタート画面でどちらのモードを使うか選択する。

**モード1：実在の結晶**
代表的な実在分子を使用する：
- C3v → アンモニア（NH₃）
- Td → メタン（CH₄）
- Oh → 六フッ化硫黄（SF₆）
- C2v → 水（H₂O）
- D6h → ベンゼン（C₆H₆）
- その他の点群にも対応する実在分子を用意する

**モード2：教科書的な架空配置**
- 実在分子にこだわらず、対称性がわかりやすい点配置を使用
- 例：C3v なら正三角形の頂点に原子を置き、中心に別の原子を配置
- 視覚的に対称性が直感的にわかりやすい配置を優先する

### サイドバー（右側）

**① 分解の進捗**
- 現在どの群にいるかをフローで表示
- 例：`C3v → C3 → E`
- 現在位置をハイライト

**② 商群選択リスト（メイン操作）**
- 現在の群に対して選べる既約な商群をボタンリストで表示
- 例：`Cs（鏡映）[1]`、`C3（3回回転）[2]`、`C2（2回回転）[3]`
- ボタンにはキーボードショートカット番号を表示
- ホバー時：対応する対称要素を3Dビューアで強調表示
- 現在の群の正規部分群として成立しない商群はグレーアウト
  - ホバーで「この操作はこの群では適用できません」と表示

**③ 見つけた組成因子**
- これまでに取り除いた商群の一覧を順番に表示
- 最終的に E に到達したら「完成」と表示し、組成因子の一覧をまとめて表示

### フッター

キーボードショートカットを常時表示：
- `1〜9`：商群を選択
- `Enter`：適用（アニメーション再生＋構造更新）
- `H`：ヒントを表示（選べる商群のうち一つをハイライト）
- `R`：現在の点群をリセット（最初からやり直す）
- `N`：別の点群を選ぶ（スタート画面へ戻る）

```
1. スタート画面
   ├── モード選択：「実在の結晶」or「教科書的な架空配置」
   └── 点群を選ぶ（例：C3v）
        ↓
2. メイン画面
   ├── 3Dビューアに結晶構造が表示される（対称要素は非表示）
   └── サイドバーに商群リストが表示される

3. 商群の選択
   ├── キーボード数字キーまたはボタンで商群を選ぶ
   └── ホバー中：対応する対称要素が3Dビューアで強調表示される
        ↓
4. 適用（Enterまたはボタンクリック）
   ├── アニメーション再生（対称要素強調 → 原子の動き → 要素消去）
   ├── 群が更新される（例：C3v → C3）
   └── 商群リストが新しい群に対応したものに更新される
        ↓
5. E になるまで 3〜4 を繰り返す
        ↓
6. 完成画面
   ├── 見つけた組成因子の一覧を表示
   ├── 「別の点群を試す」ボタン
   └── 「同じ点群を別の順で試す」ボタン
```

不明な点があれば実装前に質問すること。

- Enter で適用
- H でヒント
- R でリセット

### フェーズ9：統合・スタート画面・完成画面

## 将来の拡張：空間群への対応

最終目標は230種類の空間群への対応。
点群実装の段階から以下の設計方針を守ること。

### 点群と空間群の違い

| | 点群 | 空間群 |
|---|---|---|
| 種類 | 32種類 | 230種類 |
| 対称操作 | 回転・鏡映・反転・回反 | ＋並進・らせん軸・映進面 |
| 構造 | 1点を中心に対称 | 周期的な格子構造（単位格子） |
| 視覚化 | 分子1個でOK | 単位格子＋周期境界が必要 |
| 分解の考え方 | 商群による組成列 | 空間群→点群（因子群）→組成列 |

### フェーズ1のデータ構造設計で必ず守ること

空間群を見据えて、最初から拡張できる設計にすること。

**対称操作の表現**
対称操作を以下の形式で統一的に表現すること：
```python
# Seitz記法で統一する
# (R | t)  R: 回転行列（3x3）, t: 並進ベクトル（3次元）
# 点群の場合は t = (0, 0, 0) で表現
{
    "rotation": [[1,0,0],[0,1,0],[0,0,1]],  # 3x3回転行列
    "translation": [0, 0, 0],               # 並進ベクトル（点群は常に0）
    "type": "rotation",                     # rotation / mirror / inversion / rotoinversion / screw / glide
    "order": 3,                             # 回転の位数
    "axis": [0, 0, 1],                      # 対称軸または法線ベクトル
}
```

**群のデータ構造**
点群・空間群を同じインターフェースで扱えるよう抽象化すること：
```python
class SymmetryGroup:
    name: str                    # 例："C3v", "P6mm"
    group_type: str              # "point" または "space"
    operations: list             # 対称操作のリスト（Seitz記法）
    lattice: dict | None         # 点群はNone、空間群は格子定数
    normal_subgroups: list       # 正規部分群のリスト
    quotient_groups: list        # 既約な商群のリスト
```

**計算モジュールの抽象化**
以下の関数を点群・空間群の両方で使えるよう共通インターフェースで実装すること：
```python
def get_available_quotients(group):
    """現在の群に対して選べる既約な商群を返す"""

def apply_quotient(group, quotient):
    """商群を選んだとき、対応する正規部分群を返す"""

def apply_operation(coords, operation):
    """座標群に対称操作を適用して動かした後の座標を返す（アニメーション用）"""
```

### 3Dビューアの拡張性

Three.jsのシーン設計を最初から以下に対応できるようにすること：
- 単位格子（ユニットセル）の枠線表示
- 周期境界条件（格子が繰り返す表示）
- らせん軸（らせん状の矢印）
- 映進面（矢印付きの半透明板）

### データ形式

将来的にCIF形式（Crystallographic Information File）を読み込めるようにする。
実在の結晶データをCIF形式のデータベース（COD・ICSDなど）から直接取得できるようにする。
フェーズ3の結晶構造データは、CIF形式に変換しやすい内部フォーマットで持つこと。

### 空間群拡張のロードマップ（点群完成後）

```
フェーズ10：空間群データ構造
└── 230種類の空間群をSeitz記法で定義
    ├── らせん軸（2₁, 3₁, 4₁ など）
    └── 映進面（a, b, c, n, d）

フェーズ11：空間群の分解アルゴリズム
└── 空間群 → 点群（因子群として）
    └── 点群 → 組成列（既存ロジックを流用）

フェーズ12：単位格子の3D表示
└── 格子枠線・周期境界・原子の繰り返し表示

フェーズ13：空間群のアニメーション

    映進面：鏡映しながら並進する動き

フェーズ14：CIF形式の読み込み
└── 実在の結晶データを外部データベースから取得
```
